# Audit erreurs 500 potentielles — 2026-04-04

## Méthode
- Lint PHP global (`php -l`) sur tous les fichiers `.php`.
- Vérification des erreurs runtime existantes dans `public/error_log`.
- Relecture ciblée des points d’entrée et services critiques (DB / settings / vues admin).

## Résultats

### 1) Erreur 500 **confirmée** (bloquante)
- **Fichier** : `admin/views/partials/sidebar.php`
- **Symptôme** : erreur de parsing PHP (`unexpected end of file`).
- **Cause** : une boucle `foreach` est ouverte puis jamais fermée (`endforeach` manquant), ce qui provoque une erreur fatale au rendu de la vue.
- **Impact** : chargement de l’admin potentiellement en HTTP 500 dès inclusion de cette vue.

### 2) Erreurs SQL fréquentes observées (risque de pages partielles / panne fonctionnelle)
- **Fichier de logs** : `public/error_log`
- **Symptôme** : répétition de `SQLSTATE[42S22]: Unknown column 'setting_value' in 'SELECT'`.
- **Contexte** : les helpers de settings lisent `setting_value`.
- **Analyse** : probablement un écart de schéma entre la DB en place et les migrations attendues.
- **Impact** : pas forcément 500 immédiat ici (exceptions capturées dans les helpers), mais fort risque de dysfonctionnement global du back-office et de futures erreurs fatales dans du code non protégé.

### 3) Risques 500 techniques (préventif)
- **Connexion DB indisponible / variables d’environnement manquantes** : la couche DB renvoie explicitement un `http_response_code(500)` en mode non-debug.
- **Fonctions/extensions PHP requises absentes** (ex. `mbstring` pour `mb_encode_mimeheader`) : peut générer des erreurs fatales selon la config serveur.

## Priorité de correction
1. Corriger immédiatement la vue `sidebar.php` (bloquant, fatal).
2. Aligner le schéma SQL de `settings` avec les migrations (`004_settings_centralization.sql`) en environnement cible.
3. Vérifier les extensions PHP requises et les variables d’environnement en préprod/prod.
